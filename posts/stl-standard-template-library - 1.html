STL(Standard Template Library) - Part 1

January 9th 2014

STL or standard template library are set many useful libraries for c++ their main purpose is to make development time fast as many common data structures like stack, list, binary trees as well as some common algorithm like sorting, heaps, hashing are available there.

Containers

As the name indicates a container is collection of objects of same type for eg an int array is a collection of several int's in the same way there are many different containers available in stl like linked lists, heaps, binary trees, set etc we will explore them one by one.

There are basically 2 types of containers

  1. Sequences
    1. User controls the order in which elements are inserted in them
    2. eg:- vector, list, deque
  2. Associative Containers
    1. Here the control of elements is not in user's control the order is determined by some fixed rules which are different for each container and to access the element we have to use a key.
    2. eg:- set, multiset, map, multimap

Vector

Vectors are simply dynamic arrays as we know to declare an array we have to specify it's size for eg:- int a[10] declares an array of size 10 int *a = new int[10] creates an array of size 10 at run time but in some cases we don't know the size of our input int those cases vectors are very useful as they automatically grow in size upon adding elements to them.To use vectors you have to include it by #include<vector>

Here are 3 ways to initialize a vector

vector<int> v1;
vector<int> v2(x);
vector<int> v3(x, 1);

In the first declaration we just declare a vector of int's, in the second declaration we declare a vector of size x and in third declaration we declare a vector of size x with all it's elements set to 1 by default.

First method is used when you don't know the size of the input here is a program using first method

#include <vector>
#include <cstdio>

int main() {
    int i;
    vector<int> v;
    while(scanf("%d", &i) == 1)
        v.push_back(i);
    return 0;
}

Here in the above example we don't know size of input so we simply declare an empty vector to and an element at the end of vector use push_back(). v.size() will give you the size of the vector then to access individual elements we can simply use v[index] vectors index also start from 0 and go till v.size() - 1.Below is an example the prints an entire vector after reading it.

#include <vector>
#include <cstdio>

int main() {
    int i;
    vector<int> v;
    while(scanf("%d", &i) == 1)
        v.push_back(i);
    i = 0;
    while(i<v.size()-1) 
        printf("%d\n", v[i]);
    return 0;
}

Although the method described above is not the best way to print all the elements of vector we will look into a better method by using iterators later.

So the question is does a vector reallocate it's memory every time we add an element no when a vector is declared it allocates some fixed amount of memory then keeps on adding element till this is full when this memory gets full it reallocates double the amount of it's current memory that way it keeps on going.
